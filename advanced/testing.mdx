Docrawl has a comprehensive test suite with over 295 unit tests covering ~60% of the codebase. This guide explains how to run tests, understand test structure, and write new tests.

## Running Tests

### Basic Test Execution

<CodeGroup>

```bash All Tests
# Run all tests with verbose output
pytest -v
```

```bash With Coverage
# Run tests with coverage report
pytest --cov=src --cov-report=term-missing
```

```bash Coverage HTML Report
# Generate an HTML coverage report
pytest --cov=src --cov-report=html
# Open htmlcov/index.html in your browser
```

```bash Specific Test File
# Run tests in a specific file
pytest tests/api/test_routes.py -v
```

```bash Specific Test Function
# Run a single test function
pytest tests/api/test_routes.py::test_create_job -v
```

</CodeGroup>

### Test Markers

Tests can be filtered using pytest markers:

```bash
# Run only unit tests
pytest -m unit

# Run only integration tests
pytest -m integration

# Run only async tests
pytest -m asyncio

# Skip slow tests
pytest -m "not slow"
```

## Test Configuration

The test suite is configured in `pytest.ini`:

```ini pytest.ini
[pytest]
testpaths = tests
python_files = test_*.py
python_classes = Test*
python_functions = test_*
asyncio_mode = auto

# Coverage settings
addopts =
    --verbose
    --color=yes
    --cov=src
    --cov-report=term-missing
    --cov-report=html
    --cov-branch
    --cov-fail-under=60
    -ra

# Markers for categorizing tests
markers =
    unit: Unit tests for individual functions
    integration: Integration tests for combined functionality
    slow: Tests that take longer to run
    asyncio: Tests that use async/await
```

<Note>
  The current coverage threshold is **60%** with a target of **65%**. Tests will fail if coverage drops below 60%.
</Note>

```
├── crawler/
│   ├── test_discovery.py    # Sitemap and crawl discovery
│   ├── test_filter.py       # URL filtering logic
│   ├── test_robots.py       # robots.txt parsing and caching
│   └── test_url_filter.py   # filter_urls(), language matching (38 tests)
├── scraper/
│   ├── test_markdown.py     # html_to_markdown(), chunk_markdown() (29 tests)
│   ├── test_structured.py   # Structured output format tests
│   ├── test_converters.py   # HTML→Markdown converter plugins
│   ├── test_cache.py        # Page cache with TTL
```

## Shared Fixtures

Common fixtures are defined in `tests/conftest.py`:

```python
@pytest.fixture(scope="session")
def event_loop():
    """Create event loop for async tests."""
    loop = asyncio.get_event_loop_policy().new_event_loop()
    yield loop
    loop.close()

@pytest.fixture
def sample_urls():
    """Sample URLs for testing."""

        "https://example.com/",
        "https://example.com/page1",
        "https://example.com/api/reference",
```

## Writing Tests

### Unit Test Example

```python tests/crawler/test_filter.py
import pytest
from src.crawler.filter import filter_urls

def test_filter_urls_same_domain():
    """Test that only same-domain URLs are kept."""
    base_url = "https://example.com/docs"
    urls = [
        "https://example.com/docs/guide",
        "https://other.com/page",  # Different domain
    ]
    
    result = filter_urls(urls, base_url)
    
    assert len(result) == 1
    assert result[0] == "https://example.com/docs/guide"

@pytest.mark.asyncio
async def test_filter_urls_with_language():
    """Test language variant filtering."""
    base_url = "https://example.com/en/docs"
    urls = [
        "https://example.com/en/guide",
        "https://example.com/es/guide",  # Wrong language
    ]
    
    result = filter_urls(urls, base_url, language="en")
    
    assert len(result) == 1
    assert "/en/" in result[0]
```

### Integration Test Example

```python tests/api/test_routes.py
import pytest
from fastapi.testclient import TestClient
from src.main import app

client = TestClient(app)

@pytest.mark.integration
def test_create_job():
    """Test job creation endpoint."""
    payload = {
        "url": "https://example.com/docs",
        "output_path": "/data/test-output",
        "max_depth": 3,
    }
    
    response = client.post("/api/jobs", json=payload)
    
    assert response.status_code == 200
    data = response.json()
    assert "job_id" in data
    assert data["status"] == "pending"
```

## CI/CD Integration

Tests run automatically on GitHub Actions for:

- Push to `main` or `feat/**` branches
- Pull requests targeting `main` or `feat/**`
- Manual workflow dispatch

### Test Workflow

The CI pipeline (`.github/workflows/test.yml`) runs tests on:

- **Python 3.11** and **Python 3.12**
- Ubuntu latest
- 20-minute timeout

<Steps>
  <Step title="Setup">
    - Checkout code
    - Set up Python
    - Cache pip packages
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements-dev.txt
    playwright install --with-deps chromium
    ```
  </Step>

  <Step title="Run tests">
    ```bash
    pytest --cov=src --cov-report=xml --cov-report=term-missing
    ```
  </Step>

  <Step title="Upload coverage">
    Coverage reports are uploaded to [Codecov](https://codecov.io/gh/plater7/docrawl)
  </Step>
</Steps>

- Data directories and Docker files

## Testing Best Practices

<Steps>
  <Step title="Test isolation">
    Each test should be independent and not rely on the state from other tests.
  </Step>

  <Step title="Use fixtures">
    Leverage pytest fixtures for common setup and teardown logic.
  </Step>

  <Step title="Mock external dependencies">
    Use `unittest.mock` or `pytest-mock` to mock external APIs, databases, and file I/O.
  </Step>

  <Step title="Test edge cases">
    Include tests for error conditions, empty inputs, and boundary values.
  </Step>

  <Step title="Async testing">
    Use `@pytest.mark.asyncio` for testing async functions with `asyncio_mode = auto`.
  </Step>

  <Step title="Descriptive names">
    Use clear, descriptive test names that explain what is being tested.
  </Step>
</Steps>

## Common Test Patterns

### Testing API Endpoints

```python
from fastapi.testclient import TestClient

def test_endpoint_returns_200():
    response = client.get("/api/health")
    assert response.status_code == 200
    assert response.json()["status"] == "healthy"
```

### Testing Async Functions

```python
@pytest.mark.asyncio
async def test_async_function():
    result = await some_async_function()
    assert result is not None
```

### Mocking External Services

```python
from unittest.mock import patch, AsyncMock

@pytest.mark.asyncio
@patch("src.llm.client.httpx.AsyncClient.post")
async def test_llm_call(mock_post):
    mock_post.return_value = AsyncMock(
        status_code=200,
        json=lambda: {"response": "test"}
    )
    
    result = await generate_text("prompt")
    assert result == "test"
```

## Troubleshooting Tests

### Tests failing locally but passing in CI

- Ensure you have the correct Python version
- Check that all dependencies are installed: `pip install -r requirements-dev.txt`
- Clear pytest cache: `pytest --cache-clear`

### Async test errors

- Verify `asyncio_mode = auto` is set in `pytest.ini`
- Use `@pytest.mark.asyncio` decorator on async test functions
- Check that fixtures returning async context managers are properly awaited

### Coverage not updating

- Delete `.coverage` and `htmlcov/` directories
- Run: `pytest --cov=src --cov-report=html` again
